# 评测方案与样例

## 自动化回归

运行：

```bash
python -m pytest -q
```

覆盖重点：

- 前端自定义商品类型、卖点、使用场景和目标人群能传到后端。
- 商品主镜头必须优先绑定真实上传素材。
- `product_presence=forbidden` 的文生视频 prompt 不得携带品牌、logo、商品类型或具体外观描述。
- 高身份一致性镜头不能静默降级成纯文生商品镜头。
- 内容审核失败后必须进入修复或 `needs_review`。
- 旧 logo 特写模板和旧模板 fallback 不再存在于运行模块。

## 人工评估维度

- 商品一致性：颜色、形状、结构、logo 和材质是否稳定。
- 带货表达：是否呈现痛点、卖点证明、使用结果或转化意图。
- 素材利用：上传素材是否进入关键商品镜头，是否被错误忽略。
- 分镜连贯性：镜头之间是否风格一致、目标明确、没有无意义动作。
- 安全边界：是否出现虚构文字、错误 logo、第二个商品或过度特效。

## 当前样例结论

最近水杯和笔记本任务显示：系统链路已能把前端信息、素材身份、prompt skill 和渲染结果串起来，但部分候选仍需要内容审核和人工复核。当前策略是优先保证可复核和不伪装成功，再逐步提高视频商业表达质量。
