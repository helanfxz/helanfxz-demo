# 演示脚本

## 目标

用 3-8 分钟展示商家从输入商品信息到获得视频草稿的端到端链路，并说明系统如何处理素材、剧本、分镜、渲染和审核。

## 建议流程

1. 打开 Demo 首页，展示商品类型、核心卖点、使用场景、目标人群、风格和自定义描述输入。
2. 上传一张或多张商品素材图，说明素材会被用于商品身份和真实外观锚定。
3. 提交任务，展示进度状态和后台链路。
4. 打开任务详情页，展示结构化需求、商品身份卡、分镜计划和每镜素材绑定。
5. 展示生成的视频候选和最终状态。
6. 展示审核或 `needs_review` 信息，说明系统不会把明显不达标的视频包装成完成态。
7. 简短说明 A/B 候选、prompt skill、内容修复和本地降级能力。

## 演示样例

- 水杯：卖点可填写便携、容量、防漏、通勤携带。
- 笔记本电脑：卖点可填写轻薄、性能、移动办公、外观质感。
