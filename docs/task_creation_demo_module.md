# 任务创建演示模块说明

## 1. 模块目标

这个模块的目标不是做完整前端，而是先让“上传素材 -> 创建任务 -> 进入任务详情 -> 自动启动 agent 工作流 -> 展示草稿计划”这条链能够被直观看到。

完成后的最小效果应该是：

1. 打开一个页面
2. 填写标题、卖点、平台、时长、风格
3. 选择一个或多个文件
4. 提交后自动进入任务详情页
5. 详情页能看到任务基础信息和上传文件名
6. 控制台能看到关键调试日志，便于确认程序运行到哪一步
7. 详情页能看到“自动工作流已经推进到哪一步”
8. 页面本身能显示当前服务实例标识，便于确认浏览器命中的是哪个进程
9. 详情页能看到素材分析、商品上下文、剧本规划、分镜草稿和最终检查结果

## 2. 为什么现在先做这个模块

当前后端已经有了任务创建的领域模型，但如果没有可见页面，就很难快速判断：

- 输入项是不是合理
- 任务创建逻辑是不是按预期工作
- 上传和任务字段是不是串起来了
- 任务创建后能不能自然衔接后续工作流

所以这一模块的意义是把“领域逻辑”变成“可点击验证的流程”。

## 3. 模块边界

这个模块负责：

- 提供一个最小页面
- 接收表单输入
- 接收上传文件
- 调用任务创建逻辑
- 将创建结果重定向到独立任务详情页
- 在创建成功后自动触发 `agent/` 目录下的视频生成工作流
- 展示当前工作流阶段
- 展示工作流输出结果
- 展示当前服务实例标识
- 输出关键调试日志

这个模块暂时不负责：

- 文件持久化存储
- 文件内容解析
- 真实素材切片
- 真实视频渲染
- 任务排队与后台异步执行

## 4. 当前设计取舍

### 4.1 先用服务端渲染，不上前端构建链

这是刻意收敛，不是偷懒。

原因：

- 当前目标是快速看到链路是否跑通
- 不是做完整前端工程
- 服务端直接回显结果，调试更短
- 代码量更小，更容易审查

### 4.2 创建任务成功后自动启动工作流，但系统边界仍然拆开

用户体验上采用自动启动，不要求用户再点一次“开始生成”。

但系统内部仍然保留两个明确事件：

1. 任务创建成功
2. 工作流启动成功
3. 工作流草稿结果生成成功

这样做的原因是后续排查问题时，可以清楚区分：

- 是创建任务失败
- 还是工作流启动失败

### 4.3 上传文件先只保留元数据

当前阶段只记录：

- 文件名
- 内容类型

这样做的原因是先验证“上传动作”和“任务创建动作”已经连上，而不是提前进入素材库设计。

### 4.4 保留 `UploadedAsset` 对象

这里保留一个单独的数据对象是有意义的，因为“上传文件元数据”已经是一个独立概念。

它和普通字符串列表不同，后续大概率会扩展更多字段，所以这里不合并成裸字典。

### 4.5 风格输入同时支持预设和自定义文本

页面保留 `style` 作为风格预设，同时增加 `custom_style_prompt` 作为用户自由描述。

这样做的原因是：

- 风格预设方便快速选择
- 自定义文本能表达更具体的创作意图
- 后续 LLM 生成剧本和分镜时，应优先参考用户自定义风格期望

### 4.6 任务创建后改为后台执行工作流

视频生成属于长耗时任务，不能让用户点击“上传并创建任务”后一直等待 HTTP 请求返回。
当前实现改为：

1. 表单提交后先创建任务并保存上传文件
2. 将任务状态推进到 `processing`
3. 启动后台线程执行 `run_video_generation_workflow(...)`
4. 立即跳转到任务详情页
5. 详情页每 3 秒自动刷新，展示当前进度、阶段说明和事件列表
6. 工作流结束后状态变为 `completed` 或 `needs_review`，页面停止自动刷新

这样可以把“任务是否提交成功”和“视频是否生成完成”拆开。
用户能先确认任务已经进入系统，也能在详情页看到后台当前走到了素材分析、剧本规划、分镜生成还是视频渲染。

### 4.7 控制台输出策略

默认只保留关键流程输出，避免控制台被函数级细节刷屏。
当前会输出：

- 任务创建请求
- 任务已创建
- 任务已提交后台工作流
- 工作流各阶段完成及耗时
- 视频模型调用或回退结果
- 工作流结束状态

如果需要查看更细的调试信息，可以设置环境变量 `AIGC_VERBOSE_LOG=1`。

## 6. 后续演进方向

这个模块完成后，后续最自然的下一步是：

1. 增加文件临时存储
2. 把上传素材挂接到真正的素材模块
3. 把当前同步工作流改成后台任务

5. 从“任务详情页”继续进入“视频渲染”和“分镜编辑”

## 7. 当前运行方式

当前版本支持直接在 `PyCharm` 里点击运行：

- 打开 [task_creation_demo_app.py](/home/helanfxz/AIGC/task_creation_demo_app.py)
- 点击运行按钮
- 程序会在 `__main__` 中自动启动 `uvicorn`
- 默认访问地址是 `http://127.0.0.1:8000`

控制台当前会输出这些关键日志：

- 访问首页
- 收到任务创建请求
- 文件元数据提取完成
- 创建任务命令对象完成
- 开始执行任务创建逻辑
- 输入校验通过
- 任务实体构建完成
- 任务写入内存仓储
- 任务创建成功
- 自动启动工作流
- 调用 agent 工作流
- 保存工作流结果
- 渲染任务详情页

页面上当前还会显示以下调试信息：

- 进程 PID
- 启动时间
- 运行实例 ID

这样即使本机有多个服务残留，也能判断浏览器当前访问的是不是最新启动的那一个。
