# Commerce Expression Strategies

本文档给商品带货视频生成系统中的 LLM 和 composer 参考，用来选择“这个商品、这批素材、这个目标，更适合用什么表达结构”。它不是代码里的 if/else 固定模板，也不是要求所有商品都套同一种三段式。

最终交给视频模型的 prompt 只能是中文自然语言画面描述，不能包含 `problem_solution_pair`、`direct_benefit_proof` 这类内部策略标签，不能出现评分、JSON、审核结论或决策推理。

## 使用原则

- 先理解素材，再选择表达结构：商品是什么、素材能证明什么、素材有哪些风险、卖点是否能被画面直接证明，比“想讲什么文案”更重要。
- 一个 5 秒镜头只承担一个清楚表达目标。多个策略可以服务一条视频的不同分镜，但不要把痛点、对比、质感、操作、结果、身份确认全部塞进同一个镜头。
- 策略不是互斥分类。LLM 可以组合使用，例如先用 `identity_material_confirm` 确认真实商品，再用 `feature_operation_demo` 展示操作，最后用 `usage_result_demo` 给结果场景。
- 视觉可证明性优先。能通过商品位置、材质、动作、结果、接触关系证明的，不要只靠字幕或台词宣称。
- 商品镜头必须承担商品身份、核心卖点和使用关系。生活氛围、人物情绪和场景装饰只能辅助，不能把商品挤成背景道具。
- 字幕和台词只能补充语义，例如命名卖点、提示适用人群、强调轻微差异；不能替代画面里的商品身份、动作完成、前后变化、材质细节和真实接触。
- 如果素材不足以视觉证明某个卖点，应换策略、降级表达或拆分镜头，而不是让最终 prompt 夸大“立刻见效”“明显变白”“强力除菌”等无法验证内容。

## 策略选择维度

LLM 选择表达策略时至少考虑以下维度：

- 商品目标：要卖功能效率、质感价格感、身份风格、结果场景、使用便利、信任来源，还是解决一个明确问题。
- 素材能力：素材是否能稳定复刻商品外观、logo/文字风险是否高、是否有可用手部动作、是否有包装/证书/材质细节、是否需要原素材首帧锚定。
- 视觉证明性：卖点能否通过画面看见，例如水杯放入侧袋、护肤品质地推开、小家电一键启动、服饰上身垂坠、笔记本接口连接外设。
- 风险边界：高精度 logo、屏幕文字、复杂结构、液体、柔性布料、手部接触、跨地点连续动作，都需要更保守的镜头结构。
- 受众语境：通勤、办公、家居、健身、旅行、母婴、礼赠、专业工作等语境会影响“结果场景”是否自然。

## 通用策略

### problem_solution_pair

用一个明确痛点和一个可见解决动作配对表达。

- 适用目标：商品确实在解决一个具体、日常、容易被画面识别的问题，例如桌面杂乱需要收纳架，通勤水杯容易晃动需要侧袋稳定，笔记本续航/接口/散热场景需要对应外设或使用状态。
- 不适用情况：商品主卖点是审美、质感、身份感、礼赠感或轻奢氛围；痛点无法被视觉证明；需要夸张前后对比才能成立；用户没有提供明确使用场景。
- 视觉证明要求：痛点必须先以画面状态出现，解决必须通过商品参与的动作或结果状态完成，前后状态要在同一逻辑空间内可理解。
- 商品镜头应该承担什么：商品要成为解决动作的核心承载物或工具，不能只在最后露出包装或 logo。
- 字幕/台词只允许补充不能替代什么：可以补充“通勤不晃”“桌面更清爽”，不能替代商品真实放入侧袋、桌面物品被收纳、线缆被整理等画面事实。
- 正例：背包侧袋原本空着且通勤动作晃动明显，细长水杯从第一帧已经稳定在弹力侧袋里，袋口包住下半部，人物步行时水杯只轻微晃动，最后停在稳定近景。
- 反例：字幕写“解决通勤喝水难题”，画面只是一个水杯在桌上旋转，没有背包、通勤、拿取或稳定关系。

### direct_benefit_proof

直接把核心利益点转成可见证据，不先铺垫痛点。

- 适用目标：卖点简单直接，观众一眼能从商品状态或使用动作理解，例如水杯容量大但不笨重，小家电启动快，护肤品质地清爽，服饰显瘦利落，笔记本轻薄便携。
- 不适用情况：利益点抽象且无法视觉化，例如“更懂你”“科技领先”；需要专业数据支撑但画面没有数据来源；商品身份复刻不稳。
- 视觉证明要求：必须有一个具体可见证据承接利益点，例如尺寸对手掌、液体流动、开合顺滑、收纳位置、表面反光、启动状态、上身轮廓。
- 商品镜头应该承担什么：镜头从开始就建立商品主体，后续动作围绕一个利益点推进，不让人物表演或场景氛围抢走证明任务。
- 字幕/台词只允许补充不能替代什么：可以写“轻巧好拿”“一按即用”，不能替代手握比例、一键启动、质地推开、布料垂坠这些画面证据。
- 正例：一只小型早餐机放在厨房台面，手指按下唯一按键，指示灯亮起，内部加热灯和蒸汽轻微出现，最后停在运行中的商品近景。
- 反例：画面是家庭成员开心吃早餐，商品在背景角落模糊出现，字幕写“早餐更高效”。

### usage_result_demo

从使用后的结果状态证明商品价值。

- 适用目标：商品价值主要体现在结果场景，而不是操作过程本身，例如收纳盒让桌面整齐，护肤品上脸后妆前服帖，服饰上身后的版型，水杯放进包侧袋后的出行状态，笔记本连接外设后的工作台效率。
- 不适用情况：结果需要长时间变化或医学/功效承诺；结果无法归因于商品；视频模型难以稳定复刻商品身份的新场景；需要展示复杂前置步骤。
- 视觉证明要求：结果状态必须从第一秒就存在或在单一动作后清楚形成，且商品仍然可见，不变成“只有生活结果没有商品”。
- 商品镜头应该承担什么：商品要和结果状态有直接空间关系，例如在桌面收纳结构中、在人物穿着上、在使用位置上、在完成后的工作台中。
- 字幕/台词只允许补充不能替代什么：可以补充“出门顺手”“桌面清爽”，不能替代商品处在结果位置和结果状态已经形成。
- 正例：新场景是办公桌，笔记本居中打开，旁边只有少量必要外设，线缆被整齐固定，人物手部轻触触控板，最后停在干净高效的桌面和笔记本清晰近景。
- 反例：前 4 秒展示办公室人群和咖啡，最后 1 秒突然闪过一台笔记本，字幕写“效率提升”。

### premium_texture_reveal

用光线、近景、接触和慢动作展示材质与质感。

- 适用目标：商品主卖点是材质、做工、表面处理、细节设计、礼赠感或高质感，例如金属笔记本外壳、雾面保温杯、丝滑护肤品质地、羊毛大衣纹理、小家电磨砂面板。
- 不适用情况：商品主要靠复杂功能动作证明；素材 logo 或文字高风险且会被近景放大；商品表面细节不稳定或素材模糊；低价快消品不需要过度奢华表达。
- 视觉证明要求：必须写清具体材质表现，如反光强弱、纹理方向、边缘倒角、柔软垂坠、膏体延展、透明度或液体光泽，不能只写“高级感”。
- 商品镜头应该承担什么：商品应占据画面主体，材质细节在光线变化或手部触碰中被看见，背景只提供干净承托。
- 字幕/台词只允许补充不能替代什么：可以补充“雾面金属”“细腻质地”，不能替代近景中的反光、纹理、触碰、延展和边缘结构。
- 正例：护肤品膏体被指腹轻轻推开在手背上，质地从半透明凝露变成薄薄水光，瓶身站在旁边保持清晰，最后停在瓶身和手背质地同框。
- 反例：画面是模糊暖光背景和一堆金色装饰，商品只露出一角，文案说“高端奢华质感”。

### comparison_contrast

用可控对比展示差异，但不制造无法证明的夸张结论。

- 适用目标：同一场景中能安全展示前后差异或两种状态差异，例如普通杂乱桌面 vs 使用收纳后的桌面，宽松未整理袖口 vs 穿好后的服饰轮廓，普通杯子放不进侧袋 vs 目标水杯稳定放入侧袋。
- 不适用情况：需要贬低竞品、展示虚构竞品 logo、医疗美容前后效果、夸张减肥/变白/治疗承诺；同镜生成两个相似商品会导致身份混乱；对比对象会抢走主商品。
- 视觉证明要求：对比必须在同一空间、同一尺度或同一动作逻辑下成立，且差异来自可见状态，不来自字幕宣称。
- 商品镜头应该承担什么：目标商品必须是对比后的主角，镜头要压住“只展示一个主商品”，对比对象可以是无品牌普通道具或前状态，不应成为第二主商品。
- 字幕/台词只允许补充不能替代什么：可以补充“整理前/整理后”“普通状态/使用后”，不能替代真实前后状态、尺寸关系或收纳结果。
- 正例：桌面左侧短暂展示线缆散乱和普通支架占位，镜头平移到右侧使用目标收纳底座后的清爽状态，目标商品和被整理物关系清楚，最后只停在目标商品一侧。
- 反例：画面生成两个不同品牌水杯并让字幕写“别人家的会漏，这款不会漏”，但没有任何真实密封动作或水迹证据。

### aspirational_lifestyle_result

用理想生活方式和结果氛围表达商品带来的状态。

- 适用目标：商品购买动机偏向身份、审美、生活方式、场景想象或长期陪伴，例如通勤杯、香薰、小家电、服饰穿搭、轻薄笔记本、礼盒护肤品。
- 不适用情况：商品身份需要强锚定但新场景容易变形；卖点必须靠功能证明；生活方式画面容易让商品消失；用户需要硬核参数或操作证明。
- 视觉证明要求：生活方式必须和商品有稳定空间关系，例如被手持、穿在身上、放在桌面中心、装在包侧袋、运行在厨房台面，而不是纯氛围空镜。
- 商品镜头应该承担什么：商品是生活方式的可见触点，人物、场景、道具只服务商品的使用情境和结果感。
- 字幕/台词只允许补充不能替代什么：可以补充“周末慢早餐”“通勤更从容”，不能替代商品在早餐台、通勤包、穿搭、办公桌里的真实存在。
- 正例：清晨窗边小餐桌上，小型咖啡机正在出液，杯子放在出液口下方，人物只在边缘伸手拿杯，最后停在运行中的商品和早餐场景。
- 反例：海边、阳光、人物奔跑、咖啡和书都很好看，但商品没有清楚出现，字幕写“把生活过成理想状态”。

### feature_operation_demo

用一个可操作动作证明具体功能。

- 适用目标：商品有明确结构或按钮、开合、折叠、调节、喷雾、加热、吸附、磁吸、拉链、屏幕、接口等功能，需要让观众看到“怎么用”。
- 不适用情况：功能动作复杂到 5 秒难以完成；涉及危险、医疗、过强承诺或高风险液体喷溅；素材没有足够结构信息；手部和商品接触容易穿模时不宜强行复杂操作。
- 视觉证明要求：动作必须有起点、接触、施力、变化、完成状态。尤其手部、按钮、铰链、盖子、接口、布料和液体要写真实接触关系。
- 商品镜头应该承担什么：镜头要清楚展示操作部位和结果状态，不让手挡住关键结构，不在动作中改写商品形状。
- 字幕/台词只允许补充不能替代什么：可以补充“一键启动”“三档调节”，不能替代按键被按下、档位变化、盖子打开、喷雾出现、接口插入等画面。
- 正例：手指先触碰小风扇底座按键再按下，指示灯亮起，扇叶开始轻微转动，商品始终立在桌面，最后停在运行状态近景。
- 反例：字幕写“三档风力一键切换”，画面只展示小风扇静止摆拍，没有按键、灯光或扇叶变化。

### identity_material_confirm

先确认这就是同一件商品，并用材质或关键结构稳定商品身份。

- 适用目标：素材商品需要强身份一致性，后续视频要进入更大胆的新场景；商品有明显外观、材质、结构、标识区域或包装形态需要先建立可信锚点。
- 不适用情况：商品外观极普通且素材不足；用户只需要泛品类概念片；高精度 logo/文字无法由视频模型稳定复刻时，不应把清晰文字当核心证明。
- 视觉证明要求：必须具体写出颜色、比例、轮廓、材质、关键结构和低风险标识处理；如果用原素材首帧，场景不能切到新地点。
- 商品镜头应该承担什么：镜头只负责确认真实商品身份和材质，不承担复杂剧情、跨地点移动或多动作卖点。
- 字幕/台词只允许补充不能替代什么：可以补充商品名或品类，不能替代商品外观复刻、材质细节、结构比例和标识区域稳定。
- 正例：第一帧保持上传素材中的浅灰直筒保温杯，桌面和光线不变，镜头缓慢推进，展示黑色平盖、窄银边、雾面杯身和小面积低对比度标识，最后停在清晰近景。
- 反例：素材是一只水杯，但新场景直接生成一个颜色相近的杯子放进背包，prompt 只写“参考图同款水杯”，没有材质、比例、盖子和标识约束。

### source_context_extension

在原素材场景中轻微延展，用低风险动作或光影保持真实感。

- 适用目标：素材首帧质量较好，商品身份和背景都值得保留；目标是让静态素材动起来、确认商品可信度或展示轻微接触。
- 不适用情况：用户目标是新生活场景结果；素材背景和目标场景冲突；需要把桌面商品放进包里、穿到身上、带到户外等跨空间变化。
- 视觉证明要求：第一帧就是素材场景，地点、承托面、商品位置、光线延续一致；动作只能是轻微推进、光影变化、手部短动作或同场景内小范围运动。
- 商品镜头应该承担什么：商品保持身份锚定，镜头证明“这是素材里的真实商品”，或完成一个同场景可实现的小动作。
- 字幕/台词只允许补充不能替代什么：可以补充“细节实拍感”，不能替代原场景连续、商品位置稳定和材质可见。
- 正例：护肤品瓶身立在原素材台面，窗边光线轻微移动，手从一侧靠近但不遮住瓶身，最后停在瓶身和材质近景。
- 反例：第一帧说使用素材桌面，2 秒后同一镜头来到商场试衣间，商品变成服饰上身效果。

### scene_fit_showcase

展示商品与目标场景、尺寸、承载位置或环境的适配关系。

- 适用目标：卖点是“适合某个场景”，例如水杯适合背包侧袋，笔记本适合咖啡店办公，小家电适合小厨房，护肤品适合梳妆台，外套适合通勤穿搭。
- 不适用情况：商品没有明确场景关系；场景过强会盖过商品；需要复杂跨地点叙事；适配关系无法被镜头看见。
- 视觉证明要求：要有商品和场景元素的空间关系、比例关系或接触关系，例如放入、摆放、穿着、悬挂、连接、收纳。
- 商品镜头应该承担什么：商品必须在适配位置上被看见，镜头要交代承载物、人物、相机和商品之间的相对位置。
- 字幕/台词只允许补充不能替代什么：可以补充“适合小户型”“通勤随身”，不能替代商品确实放在小厨房台面、背包侧袋、通勤穿搭或办公桌中的画面。
- 正例：窄小厨房台面上，小型空气炸锅靠墙摆放，旁边还有砧板和调料罐，手拉开抽屉但不移动机器，最后停在商品与小台面比例清楚的画面。
- 反例：字幕写“宿舍也能用”，画面是空泛厨房大景，商品尺寸和宿舍空间没有任何关系。

### trust_source_evidence

通过来源、包装、认证、实拍感或细节证据建立信任，但不依赖虚构文字。

- 适用目标：商品需要解决“是否真实、是否正品、是否有规格依据”的信任问题，例如护肤品包装、服饰吊牌材质、家电配件、笔记本接口细节、水杯内胆结构。
- 不适用情况：素材没有包装/配件/细节证据；需要生成清晰证书编号、检测报告、品牌授权书等高风险文字；信任点无法视觉证明。
- 视觉证明要求：证据应来自真实可拍的包装形态、配件摆放、结构细节、做工、密封、接口、吊牌轮廓等，不要求模型生成清晰可读的小字。
- 商品镜头应该承担什么：商品和证据物要同框且主次清楚，证据辅助商品，不变成文字海报。
- 字幕/台词只允许补充不能替代什么：可以补充“原装配件齐全”“细节实拍”，不能替代配件同框、包装结构、接口或材质细节。
- 正例：笔记本侧边接口近景，充电口、USB-C 口和机身倒角清晰，旁边电源适配器低调同框，屏幕文字不作为焦点。
- 反例：画面生成一张虚构认证证书和大量清晰编号，商品只在旁边小角落，字幕写“官方认证”。

### sensory_use_cue

用感官线索表达体验，但只选能被视觉模拟的线索。

- 适用目标：食品饮品、香薰、护肤品、清洁用品、服饰面料、小家电运行体验等需要表达清爽、温热、柔软、香氛、安静、顺滑。
- 不适用情况：感官卖点完全不可视；需要夸大气味、味道、医学效果；画面容易变成抽象氛围而丢失商品。
- 视觉证明要求：用蒸汽、质地延展、布料贴合、液体流动、光泽变化、低幅震动、柔和风动等可见线索承接体验。
- 商品镜头应该承担什么：商品必须是感官线索的来源或承载者，例如香薰机出雾、面霜在手背推开、衣料被手指轻捻、杯中热饮蒸汽从杯口升起。
- 字幕/台词只允许补充不能替代什么：可以补充“清爽不粘”“安静运行”，不能替代质地推开、运行状态、蒸汽、布料触感这些画面线索。
- 正例：香薰机放在床头柜，出雾口有细薄雾气缓慢上升，灯光柔和，手不遮挡商品，最后停在机器和雾气同框。
- 反例：字幕写“闻起来很治愈”，画面只有卧室空镜和虚化灯光，香薰机没有出现。

### gift_unboxing_reveal

用开箱、陈列或礼赠展示商品完整度和仪式感。

- 适用目标：礼盒、护肤套装、杯具套装、小家电配件、服饰包装、数码产品开箱等，购买动机包含送礼、完整套装、到手感。
- 不适用情况：包装文字和 logo 必须高精度复刻但素材不足；开箱动作复杂或需要展示多个小件导致 5 秒混乱；商品单体卖点比包装更重要。
- 视觉证明要求：开箱动作要简单，盒盖、内托、主商品、配件位置要稳定；如果无法稳定生成文字，就把文字当低对比度块处理。
- 商品镜头应该承担什么：主商品必须在开箱后成为视觉中心，包装只承担信任和仪式感，不抢走主商品。
- 字幕/台词只允许补充不能替代什么：可以补充“送礼体面”“套装齐全”，不能替代盒内陈列、主商品出现、配件关系清楚。
- 正例：礼盒盖被手轻轻掀开，内托里是一支护肤品瓶身和一只小样，瓶身颜色与素材一致，最后停在主商品居中、包装低调的近景。
- 反例：镜头快速展示五六个包装盒、贺卡、花束和品牌大字，主商品没有稳定清晰出现。

### routine_integration

把商品放进日常流程中的一个自然节点。

- 适用目标：商品价值来自“顺手、常用、不打扰流程”，例如早起护肤、出门装包、办公室开会、运动补水、晚间清洁、小家电早餐准备。
- 不适用情况：需要强功能证明或强前后对比；流程太长；商品只是流程中可有可无的背景道具。
- 视觉证明要求：只选一个流程节点，明确前后动作边界，例如拿起、放下、按下、穿上、收进、摆到固定位置，不能跨多个地点完成一整天故事。
- 商品镜头应该承担什么：商品要被人物真实使用或稳定放在流程关键位置，镜头证明它自然融入日常。
- 字幕/台词只允许补充不能替代什么：可以补充“出门前顺手带上”，不能替代手拿、包侧袋、桌面位置、穿着状态或小家电运行。
- 正例：出门前玄关，人物把已装好水的杯子放进背包侧袋并确认稳定，动作完成后背包仍在玄关同一位置，最后停在侧袋近景。
- 反例：5 秒内从起床护肤、做早餐、出门、到办公室、运动补水连续切换，商品在每个场景都变成不同状态。

## 组合方式

- 商品身份风险高时，优先用 `identity_material_confirm` 或 `source_context_extension` 做第一镜，后续再进入新场景。
- 功能明确且可操作时，用 `feature_operation_demo` 或 `direct_benefit_proof`，不要强行写痛点故事。
- 商品主打颜值、材质、礼赠或风格时，用 `premium_texture_reveal`、`gift_unboxing_reveal`、`aspirational_lifestyle_result`，痛点解决只作为可选补充。
- 商品主打场景适配时，用 `scene_fit_showcase` 或 `routine_integration`，让商品和承载位置从第一秒建立关系。
- 商品主打结果状态时，用 `usage_result_demo`，但要保证商品仍清楚可见并能解释结果来源。
- 商品主打差异时，用 `comparison_contrast`，但避免竞品攻击、虚构品牌、医疗美容前后效果和第二主商品混乱。

## 不应写进最终 prompt 的内容

以下内容只能在内部规划、选择和审核阶段存在：

- 策略 id：`problem_solution_pair`、`direct_benefit_proof`、`usage_result_demo` 等。
- 决策理由：例如“因为该商品视觉证明性较高，所以选择直接利益证明”。
- 评分字段、风险字段、失败标签、JSON、YAML、表格和 Markdown 标记。
- “本镜头采用某策略”“这是痛点-解决结构”“本 prompt 已通过审核”等自我说明。
- 夸大且画面不可证明的功效承诺、虚构检测报告、虚构认证文字、贬低竞品的字幕。

最终 prompt 应把策略选择翻译成可拍摄的画面：商品外观、场景、位置、动作、接触关系、时间推进、最后一帧状态和必要边界。
