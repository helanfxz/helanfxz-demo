# 反模式库

这些反模式供 composer 和审核器在组装商品视频 prompt 时读取。它们不是最终给视频模型的文字；最终 prompt 应只保留明确、自然、无冲突的中文分镜描述，不能包含反例、内部字段、JSON、评分或审核结论。

## 1. 首帧素材 + 新地点冲突

### 反例
```text
第一帧完全使用素材图里的白色桌面和水杯，镜头拉远后变成清晨地铁站，水杯已经在通勤者背包侧袋里。
```

### 为什么错
同一个 5 秒镜头同时要求“素材首帧不变”和“新地点新承载位置”。视频模型常见失败是把桌面、地铁站、背包侧袋混成一个空间，或者在桌面和侧袋各生成一个商品。

### 正确改法
```text
这是新的独立分镜，不使用素材图作为第一帧。根据素材理解商品外观：浅灰磨砂直筒保温杯、黑色平盖、窄银边、小面积低对比度标识。新场景是清晨地铁站入口，商品从第一帧开始已经放在深灰双肩包右侧弹力袋里，镜头只跟拍侧袋通勤移动。
```

或改用原场景延展：
```text
第一帧就是素材图里的白色桌面场景。商品仍在桌面上，一只手从画面右侧进入并拿起商品，整个镜头不换地点，不出现地铁站和背包侧袋。
```

### 检查句
如果 prompt 同时出现“第一帧使用素材图/参考图不变”和“场景变成/来到/切到新地点”，必须判为冲突。

## 2. 纯自由生成错误商品

### 反例
```text
一个年轻人背着包走路，侧袋里有一个漂亮水杯，镜头展示它很适合通勤。
```

### 为什么错
没有写素材商品的颜色、形状、材质、尺寸比例和关键结构，模型会自由生成任意杯子。对商品带货视频来说，这等同于商品身份失败。

### 正确改法
```text
根据素材理解商品外观：杯身为细长直筒比例，主色是雾面浅灰，顶部是平整黑色杯盖，杯口有一圈很窄的金属银边，杯身表面没有大面积图案，只允许出现一处低对比度小标识。新场景中复刻这只杯子的外观，并让它已经放在背包侧袋里。
```

### 反例
```text
侧袋里有一个类似素材的商品，颜色和样子可以自然发挥。
```

### 为什么错
“类似”和“自然发挥”会鼓励模型改商品。素材理解阶段可以有不确定项，但最终 prompt 必须给出可执行外观。

## 3. 多个动作塞进 5 秒

### 反例
```text
0-1 秒手从桌上拿起水杯，1-2 秒打开背包侧袋，2-3 秒把水杯塞进去，3-4 秒背上包走进地铁，4-5 秒到办公室取出喝水。
```

### 为什么错
5 秒内包含拿起、开袋、插入、背包、换地点、取出、饮用。模型会省略步骤、跳切、生成第二个商品，或让手和背包关系穿模。

### 正确改法
拆成多个分镜：
```text
分镜 1：原素材场景，手从桌面拿起商品，不换地点。
分镜 2：新场景或中性近景，商品已经在背包侧袋里，镜头只展示侧袋接触关系。
分镜 3：通勤场景，商品随背包轻微晃动，最后一帧稳定近景。
```

单镜正例：
```text
这个 5 秒镜头只表现通勤者背着包向前走，商品从第一帧开始已经在背包侧袋里。0-5 秒内不拿起、不塞入、不取出、不饮用、不换地点。
```

## 4. 前几个自由铺垫和商品割裂

### 反例
```text
0-3 秒展示城市清晨、地铁人流、咖啡店门口和上班族表情，3-5 秒突然切到背包侧袋里的水杯特写。
```

### 为什么错
前半段和商品没有空间连续关系，最后出现的商品像另一个镜头硬插入。带货分镜应从第一秒就建立商品位置和承载关系。

### 正确改法
```text
0.0-1.0 秒通勤者背包侧面进入画面，商品已经在侧袋里；1.0-3.5 秒镜头跟随背包移动，同时展示城市通勤背景；3.5-5.0 秒收紧到商品近景。城市氛围存在，但不抢走商品主体。
```

### 检查句
如果前 2 秒完全没有商品或商品承载位置，且最后才出现商品，标记 `prompt_intent_lost` 或 `shot_redundant`。

## 5. 多图强行全用

### 反例
```text
同时参考图 1 的桌面角度、图 2 的手持角度、图 3 的包装盒正面、图 4 的户外背包场景，并在一个 5 秒镜头里全部呈现。
```

### 为什么错
多图可能提供商品不同角度，也可能包含互相冲突的背景、手势、包装和光线。强行全用会导致空间混乱、商品复制、包装盒残留。

### 正确改法
```text
只抽取多张素材中一致的商品特征：主色、轮廓、材质、盖子结构、尺寸比例。新分镜只使用一个地点、一个承载位置和一个动作。包装盒、手持角度、桌面背景不进入最终画面，除非当前 skill 明确需要它们。
```

### 可用句式
```text
多张素材只用于理解同一商品外观，不要求视频同时复现所有素材背景、角度、包装或手势。
```

## 6. LLM 自行评分自行通过

### 反例
```text
我判断这个 prompt 的商品一致性 5 分、物理合理性 5 分、场景无冲突 5 分，因此可以直接通过。
```

### 为什么错
LLM 自评容易把愿望当事实，尤其在 logo、动作数量、场景冲突和首帧绑定上会漏判。评分字段是内部诊断，不应作为最终通过依据，更不能出现在给视频模型的 prompt 里。

### 正确做法
使用确定性校验：
```text
检查 action_count 是否大于 1。
检查是否同时出现“素材首帧”和“新地点”。
检查 logo_text_risk 是否超过技能阈值。
检查 0-5 秒时间块是否发生在同一地点。
检查最后一帧是否仍有单一主商品。
```

### 禁止句式
```text
因为我认为没问题，所以无需人工审核。
```
```text
评分达到 5 分，视频模型应严格成功。
```

## 7. 自动修复无限重渲染

### 反例
```text
如果生成失败，就让系统自动改 prompt 再渲染，直到成功为止。
```

### 为什么错
无限重渲染会消耗资源，且可能在没有新信息时反复制造相同失败。常见循环包括：商品一直变形、logo 一直错误、场景一直混合、第二个商品一直出现。

### 正确做法
设置有界修复：
```text
同一失败标签最多自动修复 1-2 次。若连续出现 `logo_changed`，降低 logo 要求或切换图片锚定策略；若连续出现 `scene_conflict`，删除首帧绑定或改回原场景技能；若连续出现 `second_product_generated`，强化单一商品和空间边界。超过重试上限后交给主 agent 或人工审核。
```

### 终止条件
- 同一 prompt 家族连续两次出现同一高风险失败标签。
- 修复方案需要改变用户意图，例如从新地点改回原地点。
- 商品属于高精度 logo 或复杂结构，但当前 skill 是文字复刻。
- 失败来自素材信息不足，无法确定商品真实外观。

## 8. 互相冲突的禁止字段塞给视频模型

### 反例
```text
不要出现手，不要没有手；不要使用素材首帧，但第一帧必须和素材完全一致；不要切换地点，但最后来到办公室。
```

### 为什么错
视频模型接收的是自然语言，不会像程序一样解析复杂冲突约束。互相矛盾的禁止句会稀释真正重要的边界。

### 正确改法
用正向、单一的可拍摄描述替代冲突禁止：
```text
第一帧就是新的通勤场景，商品已经在背包右侧袋里。整个 5 秒镜头都在同一条写字楼外人行道上完成，人物只向前走，最后停在商品近景。
```

## 9. 把内部控制信息直接暴露给视频模型

### 反例
```text
identity_risk=2, logo_text_risk=3, action_count=1, forbidden_if=false，请生成通过校验的视频。
```

### 为什么错
视频模型不需要也不可靠理解内部字段。复杂 JSON、评分、阈值、审核结论会占用 prompt 空间，并可能让模型忽略真正的画面描述。

### 正确改法
```text
竖屏 9:16，写实商品短视频。新场景是清晨写字楼外人行道，商品是一只浅灰磨砂直筒保温杯，黑色平盖，窄银边，小面积低对比度标识，已经放在深灰双肩包右侧弹力袋里。
```

## 10. 正确 prompt 的最低标准

一个合格的商品分镜 prompt 至少应满足：
- 一个镜头只服务一个分镜目标。
- 一个 5 秒镜头只包含一个连续动作。
- 新场景和素材首帧绑定不能同时存在。
- 商品从第一秒就有明确位置，不能最后突然出现。
- 商品外观由素材理解转成可执行文字，不能让模型自由发挥。
- 多图素材只抽取一致商品特征，不强行复现所有图。
- 最终给视频模型的是自然语言，不是 JSON、分数、内部推理或自评结论。

## 11. 泛生活镜头没有商业证明

### 反例
```text
清晨城市街道，年轻人走过咖啡店门口，阳光很好，字幕写“轻松开启一天”。
```

### 为什么错
画面本身没有商品、承载位置、使用结果或卖点证据。即使字幕听起来像带货，也无法证明商品价值。之前自由生成铺垫镜头经常失败在这里：画面好看，但和后续商品镜头割裂。

### 正确改法
把生活场景改成能承接商品价值的可见关系：
```text
清晨办公楼入口，通勤者背着深灰色双肩包从玻璃门旁经过，背包右侧弹力袋里已经稳定放着同一款水杯，袋口包住杯身下半部，商品随步伐轻微晃动但不滑出。
```

### 检查句
如果删掉字幕后，画面看不出商品解决了什么、证明了什么或适合什么场景，标记 `scene_not_expressed` 或 `prompt_intent_lost`。

## 12. 痛点没有被商品镜头解决

### 反例
```text
第一镜桌面很乱，第二镜商品在白底棚拍中旋转，字幕写“让桌面更整洁”。
```

### 为什么错
痛点和商品镜头之间没有可见因果。观众看不到商品如何让桌面变整洁，只看到两个互不相干的画面。

### 正确改法
```text
第一镜展示线缆和小物件占满桌面一角。第二镜硬切到同一类桌面结果状态：目标收纳底座已经放在桌面右侧，线缆从底座后方整齐引出，手机和钥匙分别放入对应位置，最后停在收纳关系清楚的近景。
```

### 检查句
痛点镜头和商品镜头必须共享同一类问题、同一类空间逻辑或同一类结果指标。否则不要使用 `problem_solution_pair`，改用直接卖点、质感、场景适配或结果展示结构。

## 13. 只展示商品存在，不展示商品价值

### 反例
```text
商品完整出现在桌面上，镜头缓慢推近，字幕写“核心卖点看得见”。
```

### 为什么错
商品身份被展示了，但观众不知道它为什么值得买。身份确认镜头可以存在，但不能让整条视频都停留在“商品在画面里”。

### 正确改法
根据商品目标选择一个可视觉证明的结构：
```text
如果主打质感：让自然光扫过雾面金属边缘，手指轻触倒角后停住。
如果主打操作：手指先接触按键再按下，指示灯亮起，商品进入运行状态。
如果主打场景适配：商品从第一帧就在目标承载位置，比例和接触关系清楚。
```

### 检查句
每个商品镜头都要回答“这一镜证明了什么”。如果答案只有“商品出现了”，只能作为身份确认镜，不能作为卖点证明镜。

## 14. 强行套痛点-解决

### 反例
```text
为了卖一支香水，先拍用户焦虑烦躁，再拍香水出现，字幕写“解决你的坏心情”。
```

### 为什么错
不是所有商品都适合痛点-解决。主卖点是质感、审美、礼赠、身份感或生活方式时，硬造痛点会显得假，而且画面很难证明“解决”。

### 正确改法
```text
香水可以用 premium_texture_reveal 或 aspirational_lifestyle_result：瓶身放在清晨梳妆台，透明玻璃边缘有细腻反光，手指轻触瓶盖后停住，最后停在瓶身、喷头和柔和光线同框的质感近景。
```

### 检查句
策略选择应由用户目标、素材能力、风险和视觉证明性决定。不要把所有商品默认改成痛点-解决，也不要把所有商品默认改成通勤/办公故事。
