# Prompt 块规范

本规范定义最终交给视频模型的中文自然语言 prompt 结构。它面向 composer：composer 可以读取技能、评分、失败标签和槽位，但视频模型只应接收可拍摄、无冲突、自然连贯的画面描述；内部槽位名、JSON、评分和审核说明必须在填槽前后被移除，不能作为最终 prompt 的一部分。

## 总原则
- 最终 prompt 必须是中文自然语言，不输出复杂 JSON、YAML、表格、风险分、通过结论、内部字段名或审核规则。
- prompt 应告诉视频模型“拍什么、在哪里、商品是什么、怎样运动、最后停在哪里”，而不是告诉它“评分多少、禁止条件是否通过”。
- 避免互相冲突的禁止字段。不要同时写“第一帧使用素材图”和“新地点新时间”；不要同时写“不要手”和“手拿起商品”；不要同时写“不换地点”和“最后来到办公室”。
- 用正向描述优先，必要边界放在最后一句简短说明。边界越少越具体越好。
- 对新场景文字复刻类技能，必须明确：根据素材理解商品外观，但这是新的独立分镜，不把素材图作为第一帧。
- 对原素材场景延展类技能，必须明确：第一帧就是素材场景，地点不变，不切到新空间。

## 推荐长度
- 低风险静态展示：600-1000 个中文字符。
- 普通 5 秒商品动作：900-1400 个中文字符。
- 复杂接触、手部、承载、柔性材料或遮挡关系：1000-1400 个中文字符。

长度不足时优先补商品复刻特征、空间边界、接触物理和最后一帧；不要补空泛广告词。长度过长时删除品牌口号、情绪形容、重复禁止句和内部解释。

## 标准块顺序

### 1. 画幅和风格
说明画面比例、真实感、广告类型和镜头气质。

正例：
```text
竖屏 9:16，写实商品带货短视频，真实清晨通勤广告质感，镜头稳定、光线自然、细节清楚。
```

反例：
```text
高端大气上档次，电影感拉满，氛围炸裂。
```
问题：缺少可拍摄信息。

### 1.5 商业表达目标
在最终 prompt 中用自然语言写清“这一镜画面要证明什么”，但不要暴露内部策略 id。

正例：
```text
这个镜头要用画面证明商品适合通勤随身携带：商品从第一帧就稳定在背包侧袋里，随步伐轻微晃动但不滑出。
```

正例：
```text
这个镜头只负责展示材质质感：自然光扫过雾面杯身和窄银边，手指轻触杯身后停住。
```

反例：
```text
本镜头采用 problem_solution_pair 策略，商业目标得分 5 分。
```
问题：暴露内部策略和评分，不能给视频模型。

反例：
```text
这一镜表达高级感和好用。
```
问题：没有说清高级感或好用由哪个可见画面证明。

### 2. 镜头来源边界
必须二选一：原素材延展，或新场景文字复刻。不要混写。

原素材延展正例：
```text
第一帧就是素材图中的真实画面，桌面、背景、商品位置和光线都顺着素材场景延展，整个 5 秒镜头不换地点。
```

新场景文字复刻正例：
```text
这是新的独立分镜、新地点、新时间，不使用素材图作为第一帧，也不接续上一镜的桌面、房间、手部动作或包装画面。
```

反例：
```text
第一帧完全参考素材图，但场景变成地铁站。
```

### 3. 场景边界
写清地点、时间、光线、背景，只保留一个空间。

正例：
```text
场景是清晨写字楼外的人行道，玻璃幕墙反射柔和天光，远处有虚化的通勤人群和绿植，整个镜头都发生在这条人行道上。
```

反例：
```text
从家里出发，经过地铁，最后到办公室。
```
问题：多个地点塞进一个镜头。

### 4. 商品外观与文字/logo处理
商品外观必须具体。对文字复刻类技能，先写“根据素材理解”，再写可复刻特征；不要让视频模型自己解析素材。

正例：
```text
根据素材理解商品外观：商品是一只细长直筒保温杯，雾面浅灰杯身，顶部黑色平盖，杯口有一圈很窄的银色边缘，杯身表面干净，只允许出现一处小面积低对比度标识，不生成清晰大字或虚构大 logo。
```

正例：
```text
商品是半透明浅绿色水瓶，瓶身略扁的圆柱比例，白色翻盖，盖侧有短挂环，瓶身能透出柔和液体阴影，小标签只作为模糊浅色块处理。
```

反例：
```text
商品和参考图完全一致，所有 logo 和文字必须一模一样。
```
问题：对低锚定文字复刻任务提出高精度 logo 要求。

反例：
```text
一个好看的杯子。
```
问题：商品身份不可控。

### 5. 空间布局
写商品、承载物、人物、镜头、背景的相对位置。

正例：
```text
商品从第一帧开始已经放在深灰色双肩包右侧弹力网袋里，袋口包住杯身下半部，杯身上半部露出朝向镜头；通勤者穿浅色风衣从画面左侧经过，背景人群保持虚化。
```

反例：
```text
商品出现，人物也出现，背景很好看。
```

### 6. 单一连续动作
一个 5 秒镜头只写一个动作目标。

正例：
```text
这个镜头只表现通勤者背着包向前走，侧袋里的商品随步伐轻微晃动并被镜头展示。
```

反例：
```text
拿起商品、放进包里、走进地铁、到办公室再取出来。
```

### 7. 0-5 秒时间分块
必须描述每个时间段的画面推进。时间块要延续同一地点、同一动作、同一主商品。

正例：
```text
0.0-1.0 秒：背包侧面进入画面，商品已经稳定在侧袋里，位置清楚。
1.0-2.2 秒：镜头贴近跟拍，商品随步伐轻微晃动，袋口和商品保持接触。
2.2-3.4 秒：自然光扫过商品表面，展示颜色、材质、盖子和比例。
3.4-4.4 秒：镜头轻微收紧到侧袋近景，背景继续同一地点的移动虚化。
4.4-5.0 秒：人物步伐放慢，最后一帧稳定在商品清晰近景。
```

反例：
```text
0-2 秒在厨房，2-4 秒在地铁，4-5 秒在办公室。
```

### 8. 接触关系和物理
凡是手、口袋、侧袋、桌面、铰链、柔性材料、液体、承载物参与，都要写真实接触。

正例：
```text
侧袋边缘对商品有包裹，商品底部被袋底支撑，随背包移动产生轻微惯性晃动，但不能悬空、穿过布料、跳出侧袋或自己旋转。
```

正例：
```text
手指先接触杯身再握紧，杯子被抬起时杯底离开桌面，不能隔空移动。
```

反例：
```text
商品自然移动到合适位置。
```
问题：物理关系缺失。

### 9. 相机运动
相机运动要服务商品展示，不要制造换地点错觉。

正例：
```text
镜头位于背包右后方 45 度、腰部高度近距离跟拍，后半段轻微推进到商品近景，不切换视角到别的房间。
```

反例：
```text
镜头快速环绕、穿越城市、切到多个角度。
```

### 10. 最后一帧状态
最后一帧必须是可验收的画面状态，包含商品位置和清晰度。

正例：
```text
最后一帧商品仍稳定在背包右侧袋里，占据画面中心偏右，杯盖、杯身颜色、材质和侧袋包裹关系清晰可见，画面中只有这一个主商品。
```

反例：
```text
最后效果高级，适合购买。
```

### 11. 简短边界
最后用一两句压住关键失败，不要堆叠大量互相矛盾的禁止词。

正例：
```text
整个镜头不出现第二个商品，不回到素材场景，不表现拿起、塞入、取出或饮用。
```

反例：
```text
不要乱、不要假、不要错误、不要不好看、不要不真实、不要出现不该出现的东西。
```

## 表达结构边界

内部可以有 `problem_solution_pair`、`direct_benefit_proof`、`usage_result_demo`、`premium_texture_reveal`、`scene_fit_showcase`、`feature_operation_demo` 等策略选择，但最终 prompt 不应写出这些标签。composer 必须把策略翻译成可拍摄画面：

- 痛点-解决：写清痛点状态、商品参与方式和解决后的可见状态。
- 直接卖点证明：写清商品动作或状态怎样直接证明卖点。
- 结果展示：写清商品在结果位置中如何可见，结果为什么来自商品。
- 质感展示：写清材质、光线、触碰、反光、纹理和最后近景。
- 场景适配：写清商品和承载位置、尺寸比例、接触关系。
- 操作证明：写清起点、接触、施力、变化和完成状态。

不要把结构写成固定顺序。一个视频可以组合不同结构，一个镜头只承担一个结构目标。

## 完整正例：新场景文字复刻通勤侧袋
```text
竖屏 9:16，写实商品带货短视频，真实清晨通勤广告质感，镜头稳定、光线自然、商品细节清楚。这个镜头是新的独立分镜、新地点、新时间，不使用素材图作为第一帧，也不接续上一镜的桌面、室内房间、手拿动作或包装画面。场景是清晨写字楼外的人行道，玻璃幕墙反射柔和天光，远处有虚化的通勤人群和路边绿植，整个 5 秒镜头都发生在这条人行道上。

根据素材理解商品外观：商品是一只细长直筒保温杯，雾面浅灰杯身，顶部黑色平盖，杯口有一圈很窄的银色边缘，杯身表面干净，只允许出现一处小面积低对比度标识，不生成清晰大字或虚构大 logo。新场景里复刻这只杯子的颜色、比例、材质、盖子和边缘结构。画面主体是穿浅色风衣的通勤者，背着深灰色双肩包从画面左侧经过。商品从第一帧开始已经稳定放在双肩包右侧弹力网袋里，不是手里拿着，也不是正在塞进去；袋口包住杯身下半部，杯底被袋底支撑，杯身上半部露出朝向镜头。

镜头位于背包右后方 45 度、腰部高度近距离跟拍，只表现通勤者背着包向前走，侧袋里的商品随步伐轻微晃动并被展示。0.0-1.0 秒：背包侧面进入画面，侧袋和商品一起到达画面中部，商品已经放好。1.0-2.2 秒：镜头贴近跟拍，商品随步伐产生轻微惯性晃动，侧袋边缘和杯身保持真实接触，不能悬空。2.2-3.4 秒：自然光扫过杯身，展示浅灰磨砂材质、黑色平盖、窄银边和直筒比例，背景继续同一地点的移动虚化。3.4-4.4 秒：镜头轻微收紧到侧袋近景，商品仍只有一个，手不进入画面。4.4-5.0 秒：人物步伐放慢，最后一帧商品仍稳定在右侧袋里，占据画面中心，关键外观和袋口包裹关系清晰可见。整个镜头不出现第二个商品，不回到素材场景，不表现拿起、塞入、取出或饮用。
```

## 完整正例：原素材场景延展拿起
```text
竖屏 9:16，写实商品短视频，第一帧就是素材图中的真实画面，桌面、背景、商品位置和光线都顺着素材场景延展，整个 5 秒镜头不换地点。画面中主商品是一只浅绿色半透明水瓶，略扁圆柱瓶身，白色翻盖，盖侧有短挂环，瓶身小字只作为模糊浅色标识处理。商品放在浅木色桌面右前方，背景是柔和虚化的厨房台面。

这个镜头只表现一只手把水瓶从桌面拿起。0.0-0.8 秒：商品保持在桌面原位置，镜头轻微向前推进。0.8-1.6 秒：一只手从画面右侧进入，手指先接触瓶身中部再握紧，不能隔空控制。1.6-2.8 秒：手把水瓶沿竖直略向右的路径抬起，瓶底真实离开桌面，瓶身不穿过桌面。2.8-4.0 秒：手持水瓶停在桌面上方，背景仍是同一厨房台面，不切到新地点。4.0-5.0 秒：最后一帧水瓶被手稳定握住，瓶身颜色、翻盖、挂环和半透明材质清晰可见，画面中只有这一个主商品。
```

## 最终 prompt 不应包含
以下内容可以在内部工作流存在，但不能放进最终视频 prompt：
- JSON、YAML、Markdown 表格、代码块标记。
- `identity_risk`、`logo_text_risk`、`action_count`、`forbidden_if`、`success_stats` 等内部字段。
- “我判断通过”“评分为 5”“无需审核”等自评句。
- 大段失败分析、反模式解释或决策推理。
- 互相冲突的禁止项。

## 出口校验
composer 交付最终 prompt 前检查：
- 是否写清这一镜的商业表达目标，且目标能被画面证明。
- 是否没有出现内部表达策略标签、评分或决策推理。
- 是否只有一种镜头来源：素材首帧延展或新场景文字复刻。
- 是否只有一个地点、一个主商品、一个连续动作。
- 商品外观是否具体到颜色、形状、材质、结构和比例。
- 0-5 秒时间块是否完整，且没有跨地点跳切。
- 是否有接触关系和最后一帧状态。
- 是否删除了内部字段、评分、自评、复杂 JSON 和冲突禁止句。
